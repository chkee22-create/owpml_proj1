PaperMate v6 작업 브리핑
작성일: 2026.06.04
브랜치: choi_v6_table

============================================================
1. 작업 목적
============================================================

이번 v6 작업의 핵심 목적은 분석 화면에서 문서 원본 미리보기, 채팅 입력창,
시각화 버튼, 표 생성 기능이 하나의 흐름으로 자연스럽게 동작하도록 개선하는
것입니다.

특히 PDF와 HWP/HWPX 문서를 함께 열어도 원본 미리보기가 사라지지 않도록
상태 저장 방식을 보강했고, 표 만들기 기능은 기존 분석 결과와 GPT를 활용해
문서 구조에 맞는 요약 표를 생성하도록 개선했습니다.


============================================================
2. 원본 미리보기 유지 기능
============================================================

주요 파일
- frontend/src/pages/Analysis.tsx
- frontend/src/services/api.ts
- backend/app/routers/analysis.py

주요 함수 및 역할

handleSourceFileUpload()
- 사용자가 PDF, HWP, HWPX 파일을 추가할 때 실행됩니다.
- 파일 정보를 분석 채팅 상태와 원본 미리보기 상태에 연결합니다.
- HWP/HWPX 파일은 서버 미리보기 API를 통해 PDF 형태로 변환된 화면을 띄우도록
  연결됩니다.

restoreSourceFilesFromIndexedDb()
- 페이지 새로 고침이나 다른 페이지 이동 후 다시 돌아왔을 때 원본 미리보기를
  복구합니다.
- 브라우저 localStorage만으로는 큰 파일을 안정적으로 저장하기 어렵기 때문에
  IndexedDB를 함께 사용합니다.

saveSourceFilesToIndexedDb()
- 업로드한 원본 파일을 브라우저 IndexedDB에 저장합니다.
- 새로 고침 이후에도 원본 미리보기가 사라지지 않도록 하는 핵심 저장 함수입니다.

clearSourceFileStore()
- 새 분석을 시작할 때 기존 원본 미리보기 파일을 정리합니다.
- 한 번의 새 채팅 클릭으로 채팅과 원본 미리보기가 함께 비워지도록 연결했습니다.

previewDocument()
- frontend/src/services/api.ts에 추가된 API 함수입니다.
- HWP/HWPX 문서의 미리보기를 백엔드에 요청합니다.

preview_document()
- backend/app/routers/analysis.py에 있는 백엔드 미리보기 엔드포인트입니다.
- HWP/HWPX 문서를 브라우저에서 볼 수 있는 미리보기 결과로 변환합니다.

결과
- PDF만 열었을 때뿐 아니라 PDF와 HWP/HWPX를 여러 개 함께 열어도 미리보기가
  유지됩니다.
- 새로 고침, 페이지 이동 후 복귀 상황에서도 원본 미리보기가 복원됩니다.
- 새 채팅 버튼을 한 번만 눌러도 원본 미리보기까지 함께 사라집니다.

3. 새 채팅 및 분석 리로드 연동
============================================================

주요 파일
- frontend/src/pages/Analysis.tsx

주요 함수 및 역할

handleNewAnalysis()
- 분석 채팅창을 비웁니다.
- 업로드된 원본 파일과 미리보기 상태를 초기화합니다.
- 생성된 시각화 결과를 정리합니다.
- 기존 분석 페이지 리로드 버튼의 동작과 새 채팅 버튼을 같은 흐름으로 맞췄습니다.

sourceResetVersionRef
- 원본 미리보기 초기화 중에 이전 비동기 작업이 늦게 도착해 화면을 다시 채우는
  문제를 막기 위한 버전 관리 값입니다.
- 새 분석을 누른 뒤 이전 파일 미리보기가 되살아나는 현상을 방지합니다.

결과
- 새 채팅 버튼을 누르면 분석 채팅과 원본 미리보기가 한 번에 초기화됩니다.
- 두 번 눌러야 원본 미리보기가 사라지던 문제가 해결되었습니다.

4. 채팅창 클립 메뉴 개선
============================================================

주요 파일
- frontend/src/pages/Analysis.tsx
- frontend/src/pages/styles/Analysis.styles.ts

구성된 메뉴
- 파일 추가
- 이미지 만들기
- 그래프 만들기
- 표 만들기

주요 함수 및 역할

toggleAttachmentMenu()
- 채팅창 클립 버튼을 눌렀을 때 메뉴를 열고 닫습니다.
- 버튼 아이콘은 180도만 부드럽게 회전하도록 스타일이 적용되었습니다.

handleAttachmentAction()
- 메뉴에서 어떤 항목을 눌렀는지에 따라 동작을 분기합니다.
- 파일 추가, 이미지 만들기, 그래프 만들기, 표 만들기 모두 파일 선택 창을
  열도록 연결했습니다.

handlePromptAfterFileAdd()
- 파일을 추가한 뒤 채팅 입력창에 "분석해 드릴까요?" 문구를 자동으로 넣습니다.
- 사용자는 엔터만 눌러 분석을 시작할 수 있습니다.

스타일 개선
- 파일 추가 항목 아래에 구분선을 추가했습니다.
- 파일 추가 버튼을 다른 항목보다 더 강조해 시각적 우선순위를 줬습니다.

결과
- 클립 메뉴가 현재 화면 흐름에 맞게 동작합니다.
- 각 버튼이 기존 백엔드/앱/서비스의 시각화 기능으로 연결됩니다.

5. 표 만들기 시각화 기능
============================================================

주요 파일
- backend/app/routers/visuals.py
- backend/app/services/visual_buttons/table_visual.py
- backend/app/services/analysis_pipeline.py
- backend/app/services/document_analysis.py
- backend/app/services/llm_analysis.py
- frontend/src/components/DynamicVisualizer.tsx
- frontend/src/pages/Analysis.tsx

표 컬럼
- 제목
- 중요내용
- 중요도(점수)
- 정확도(%)

주요 함수 및 역할

create_table_visual()
- 표 만들기 버튼에서 호출되는 핵심 서비스 함수입니다.
- 문서 분석 결과와 GPT 응답을 바탕으로 문서 내용을 표 형태로 정리합니다.
- 표의 가로 비율은 와이드 형태에 맞게 구성했습니다.

generate_table_from_analysis()
- 이미 생성된 분석 결과를 기반으로 표를 다시 구성합니다.
- 분석 내용과 표 내용이 서로 다르게 보이는 문제를 줄이기 위해 추가했습니다.

_document_title()
- 문서에서 제목 후보를 추출합니다.
- 파일명이 아니라 실제 문서 상단 제목을 우선적으로 사용하도록 개선했습니다.

_title_candidate_score()
- 제목 후보의 품질을 점수화합니다.
- 너무 짧거나 의미 없는 문장, 페이지 번호, 일반 안내문이 제목으로 잡히는 문제를
  줄이기 위해 사용됩니다.

_extract_sections()
- 문서 안의 큰 제목, 소제목, 본문 흐름을 찾아 표의 행으로 구성합니다.
- "요약", "서론", "본론", "결론" 같은 고정 문구를 강제하지 않고 문서마다 다른
  구조를 따라갑니다.

_summarize_section()
- 각 섹션의 본문을 표에 들어갈 수 있도록 압축합니다.
- 긴 문단을 핵심 내용 중심으로 줄여 보여줍니다.

visuals_router.create_visual()
- 프론트엔드에서 이미지, 그래프, 표 만들기 요청을 받을 때 진입하는 백엔드 라우터입니다.
- 요청 타입에 따라 적절한 시각화 서비스로 연결합니다.

DynamicVisualizer
- 백엔드에서 받은 표 데이터를 화면에 렌더링합니다.
- 제목, 안내 문구, 표 본문을 카드 형태로 보여줍니다.

결과
- 표 제목에는 문서의 실제 대문 제목이 들어갑니다.
- 표 행은 문서의 큰 제목과 소제목 흐름을 따라 생성됩니다.
- 목차내용 컬럼은 "중요내용"으로 변경되어 더 자연스럽게 표시됩니다.
- GPT 키가 있는 경우 GPT 분석을 활용하고, 키가 없는 경우 로컬 분석 결과를 기반으로
  fallback 표를 생성합니다.

============================================================

주요 파일

- backend/app/services/analysis_pipeline.py
- backend/app/services/visual_buttons/table_visual.py

주요 함수 및 역할

- 문서 핵심 내용, 제목, 섹션 요약 등을 더 정교하게 뽑는 데 활용됩니다.

run_analysis_pipeline()
- 문서 분석의 전체 흐름을 담당합니다.
- 파일 파싱, 기본 분석, GPT 분석, 시각화용 데이터 생성을 연결합니다.

create_table_visual()
- 표 생성에서도 GPT 분석 결과를 활용합니다.
- 분석 결과와 표 결과가 따로 노는 문제를 줄이기 위해 분석 텍스트 기반 표 생성을
  우선하도록 개선했습니다.

결과
- 키가 없을 때도 기본 분석은 가능합니다.
- 키를 넣으면 문서 제목 추출, 섹션 요약, 표 내용 품질이 더 좋아지도록 구성했습니다.

- root .env와 backend/.env에 같은 Google OAuth Client ID를 넣는 구조로 정리했습니다.

11. 발표용 요약
============================================================

이번 v6 작업은 PaperMate 분석 화면의 실제 사용 흐름을 안정화한 작업입니다.

첫째, 원본 미리보기가 새로 고침이나 페이지 이동 후에도 유지되도록
IndexedDB와 세션 복원 로직을 추가했습니다.

둘째, 새 채팅 버튼과 분석 리로드 동작을 연결해 한 번의 클릭으로 채팅,
시각화 결과, 원본 미리보기가 함께 초기화되도록 했습니다.

셋째, 채팅창 클립 메뉴를 파일 추가, 이미지 만들기, 그래프 만들기, 표 만들기
기능으로 확장하고 각 버튼을 기존 백엔드 시각화 서비스와 연결했습니다.

넷째, 표 만들기 기능은 문서의 실제 제목과 섹션 구조를 추출해 제목,
중요내용, 중요도, 정확도 컬럼으로 정리하도록 개선했습니다.

다섯째, Gemini 관련 코드는 제거하고 OpenAI 기반 분석과 Google 로그인 설정을
명확히 분리했습니다.

결과적으로 v6에서는 분석 화면이 새로 고침, 문서 전환, 여러 파일 조합,
시각화 생성 상황에서도 더 안정적으로 동작합니다.
